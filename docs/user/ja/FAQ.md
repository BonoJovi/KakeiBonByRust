## インストールとセットアップ

### Q1. どのプラットフォームで動作しますか？
**A**: KakeiBonは以下のプラットフォームで動作します：
- Windows 10/11（64ビット）
- macOS 10.15以降（Intel/Apple Silicon）
- Linux（主要ディストリビューション）

現時点では、ソースコードからビルドする必要があります。詳細は[セットアップガイド](SETUP_GUIDE.md)を参照してください。

### Q2. インストールに必要なシステム要件は？
**A**: 
- **CPU**: 現代的な64ビットプロセッサ
- **メモリ**: 最低2GB RAM（4GB以上推奨）
- **ストレージ**: 100MB以上の空き容量
- **開発環境** (ビルド時):
  - Rust 1.70以上
  - Node.js 18以上
  - Git

### Q3. データベースは自動的に作成されますか？
**A**: はい。初回起動時に`data/kakeibo.db`が自動的に作成されます。手動で作成する必要はありません。

## ログインと認証

### Q4. 初回起動時に何をすればよいですか？
**A**: 初回起動時は管理者ユーザーのセットアップ画面が表示されます：
1. ユーザー名を入力（3-50文字）
2. パスワードを設定（16文字以上）
3. パスワードを確認のため再入力
4. 「管理者作成」ボタンをクリック

### Q5. パスワードの要件は何ですか？
**A**: セキュリティのため、以下の要件があります：
- **最小文字数**: 16文字
- **推奨**: 大文字、小文字、数字、記号を組み合わせる
- **禁止**: 空白文字のみのパスワード

### Q6. パスワードを忘れてしまいました
**A**: 現在、パスワードリセット機能は実装されていません。データベースファイル（`data/kakeibo.db`）をバックアップした後、削除して新しい管理者を作成する必要があります。
**注意**: この操作により全データが失われます。

### Q7. 複数のユーザーを作成できますか？
**A**: はい。管理者は「ユーザー管理」画面から、管理者または一般ユーザーを追加できます。

## データ管理

### Q8. データはどこに保存されますか？
**A**: データは以下の場所に保存されます：
- **データベース**: `data/kakeibo.db`（SQLite形式）
- **暗号化**: ユーザーごとにパスワードで暗号化（AES-256-GCM）

### Q9. データをバックアップするには？
**A**: 以下のファイルをコピーしてください：
```
data/kakeibo.db
```
定期的なバックアップを推奨します。

### Q10. データを別のコンピュータに移行できますか？
**A**: はい。`data/kakeibo.db`ファイルを新しいコンピュータの同じ場所にコピーしてください。パスワードは同じものを使用する必要があります。

### Q11. データのエクスポート機能はありますか？
**A**: 現在、CSV/PDF等へのエクスポート機能は実装されていません。将来のバージョンで追加予定です。

## セキュリティ

### Q12. データは安全ですか？
**A**: はい。以下のセキュリティ対策を実装しています：
- **パスワードハッシュ化**: Argon2id（メモリハード関数）
- **データ暗号化**: AES-256-GCM
- **ユーザーごとの暗号化**: 各ユーザーのデータは個別に暗号化
- **ローカルストレージ**: データはローカルに保存され、外部送信されません

### Q13. インターネット接続は必要ですか？
**A**: いいえ。KakeiBonは完全にオフラインで動作します。インターネット接続は不要です。

### Q14. データは外部に送信されますか？
**A**: いいえ。KakeiBonはすべてのデータをローカルに保存し、外部サーバーへの送信は一切行いません。

## トラブルシューティング

### Q15. アプリケーションが起動しません
**A**: 以下を確認してください：
1. システム要件を満たしているか
2. `data`ディレクトリへの書き込み権限があるか
3. ポート競合がないか
4. エラーログを確認（コンソール出力）

詳細は[トラブルシューティングガイド](TROUBLESHOOTING.md)を参照してください。

### Q16. データベースエラーが発生します
**A**: 
1. `data/kakeibo.db`ファイルが破損していないか確認
2. ファイルの読み書き権限を確認
3. ディスク容量が十分にあるか確認
4. バックアップから復元を試みる

### Q17. ログインできません
**A**: 
- ユーザー名とパスワードを正確に入力しているか確認
- Caps Lockがオンになっていないか確認
- データベースファイルが正常か確認

### Q18. 画面が正しく表示されません
**A**: 
- ブラウザのキャッシュをクリア（Ctrl+F5）
- ウィンドウサイズを調整
- 最新版にアップデート

## 機能について

### Q19. カテゴリは何種類登録できますか？
**A**: カテゴリ数に上限はありませんが、パフォーマンスを考慮して適切な数（推奨：50以下）に抑えることをお勧めします。

### Q20. 定期的な取引の自動登録機能はありますか？
**A**: 現在は実装されていません。将来のバージョンで追加予定です。

### Q21. グラフや統計機能はありますか？
**A**: はい。集計画面で以下が利用できます：
- 月次・年次の収支サマリー
- カテゴリ別の支出分析
- （将来実装予定）グラフ表示、トレンド分析

### Q22. 複数通貨に対応していますか？
**A**: 現在は日本円（JPY）のみ対応しています。複数通貨対応は将来のバージョンで検討中です。

### Q23. スマートフォン版はありますか？
**A**: 現在はデスクトップ版のみです。モバイル対応は将来の計画に含まれています。

### Q24. 銀行口座やクレジットカードとの連携はできますか？
**A**: 現在は対応していません。手動入力のみとなります。
