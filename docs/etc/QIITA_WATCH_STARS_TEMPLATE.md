# Qiita記事 Watch/Stars リクエストテンプレート

## 📌 概要

Qiita記事の末尾に挿入する、GitHubリポジトリのWatch/Starsをお願いする文章テンプレート集です。
読者に自然にアクションを促す効果的な文章例と、実際のリンクを提供します。

---

## 🔗 実際のリンク

### Watch（通知を受け取る）
```
https://github.com/BonoJovi/KakeiBonByRust/subscription
```

### Star（ブックマーク・応援）
```
https://github.com/BonoJovi/KakeiBonByRust
```
※ページ右上の⭐️Starボタンをクリック

### Watch & Star 両方を案内する場合
```
リポジトリ: https://github.com/BonoJovi/KakeiBonByRust
- Watch: https://github.com/BonoJovi/KakeiBonByRust/subscription
- Star: ページ右上の⭐️ボタン
```

## 💡 効果的な文章のポイント

### ✅ DO（推奨）

1. **具体的なメリットを示す**
   - 「Starすると、あなたのGitHubプロフィールから簡単にアクセスできます」
   - 「Watchすれば、新機能をいち早く試せます」

2. **行動を明確にする**
   - リンクを直接提供
   - クリック場所を具体的に説明（「右上のStarボタン」など）

3. **感謝の気持ちを伝える**
   - 「あなたの⭐が励みになります」
   - 「皆様のサポートに感謝します」

4. **絵文字で視覚的に訴求**
   - ⭐（Star）、👀（Watch）、🔔（通知）などを活用

5. **読者の状況に合わせる**
   - 初心者向け記事 → シンプル型
   - 技術的な記事 → 詳細型

### ❌ DON'T（避けるべき）

1. **押しつけがましい表現**
   - ❌ "必ずStarしてください"
   - ✅ "よろしければStarをお願いします"

2. **長すぎる文章**
   - 記事本文より長いお願い文は逆効果
   - 簡潔に、要点を絞る

3. **リンクなしのお願い**
   - 必ず実際のリンクを提供する

4. **技術的すぎる説明**
   - 一般読者向けには、シンプルな説明を

## 📊 効果測定のヒント

Watch/Starsのリクエストを記事に追加した後、以下を確認すると効果が測定できます：

1. **GitHub Insights**
   - リポジトリの `Insights` → `Traffic` でアクセス元を確認
   - `Insights` → `Community` でStar数の推移を確認

2. **Qiita Analytics**
   - 記事のView数とGitHubへのクリック率を比較

3. **時系列での比較**
   - 記事公開前後でStar/Watchの増加率を比較

## 🎯 使い方

1. 上記のパターンから、記事の内容や読者層に合ったものを選ぶ
2. 必要に応じてカスタマイズ
3. Qiita記事の末尾（まとめの後）に挿入
4. 実際のリンクが正しいことを確認してから公開
