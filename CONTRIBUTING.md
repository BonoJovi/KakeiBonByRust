# Contribution Guidelines

Thank you for your interest in contributing to this repository!
Please follow the guidelines below when contributing.

## Getting Started

- Before creating an issue or pull request, please check existing issues and discussions.
- For bug reports or feature suggestions, provide as much detail as possible.

## Bug Reports

1. Describe the steps to reproduce, the expected behavior, and the actual behavior.
2. Attach screenshots, logs, and environment details (OS/browser/version, etc.) if available.

## Feature Proposals

1. Describe specifically what feature you want.
2. Explain why it's needed, including use cases.

## Pull Request Workflow

1. **Fork** the repository and create a working branch.
2. Write clear commit messages for each change.
3. Add or modify tests as needed.
4. When creating a pull request, include details about the changes, their purpose, and any related issue numbers.
5. If reviews or fixes are needed, add commits as appropriate.

## Code Style

- Follow the existing code style.
- If a linter or formatter is configured, adhere to its rules.

## Other

- For questions or discussions, please contact us via Issues, Discussions, or [administrators](https://github.com/BonoJovi).
- All participants are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md).

---

Thank you for your cooperation!
